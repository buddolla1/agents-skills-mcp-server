## Agent index

Agent files are reusable, task-focused prompt definitions that let GitHub Copilot or similar tooling invoke a specialized assistant for a specific job. Each agent captures a focused purpose, when it should be used, and the boundaries it should follow.

Why agents are useful:

- They give repeated tasks a stable, reusable operating model.
- They package domain-specific behavior into a callable unit instead of repeating long prompts.
- They improve consistency in analysis, reporting, and generated artifacts.
- They help separate narrow specialists, orchestrators, and fix-oriented flows.
- They make repository automation and team reuse easier.

Key advantages:

- Clearer task routing
- More predictable outputs
- Better reuse across repositories
- Stronger adherence to local engineering standards
- Easier automation for recurring work
