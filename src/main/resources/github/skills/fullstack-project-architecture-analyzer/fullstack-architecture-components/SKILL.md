# Full-Stack Architecture Components

Use this skill to identify and document the main application components and their responsibilities.

## When to Use This Skill

Use this skill when the user selects components or full doc and wants the component view documented.

## Prerequisites

- Source files and runtime configuration
- Project type
- Major runtime modules and boundaries

## Goal

Describe the important application components and how they relate to one another.

## Output Standard

Provide:

- Components
- Responsibilities
- Interaction notes
- Component diagram when useful

## Guardrails

- Do not invent components that do not exist.
- Do not repeat HLD or LLD without adding component-level value.

## Reporting Style

- Be factual and specific.
- Keep the component map readable.
